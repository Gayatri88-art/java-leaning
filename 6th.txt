### 🔍 Why Not Use `==` for Strings in Java?
In Java, **`==` compares object references**, not the actual contents of the strings.
String a = "yes";
String b = new String("yes");

System.out.println(a == b); // ❌ Might be false
System.out.println(a.equals(b)); // ✅ true
```

To check if **two strings have the same content**, we use `.equals()`.

---

### ✅ `.equals()` vs `.equalsIgnoreCase()`

| Method                        | Description                                     | Case Sensitive? |
| ----------------------------- | ----------------------------------------------- | --------------- |
| `str1.equals(str2)`           | Compares the exact content of both strings      | Yes             |
| `str1.equalsIgnoreCase(str2)` | Compares the content, ignoring upper/lower case | No              |

##when to use nested if, nestef else
| Concept         | When to Use                       | Example Scenario                     |
| --------------- | --------------------------------- | ------------------------------------ |
| **Nested if**   | One condition depends on another  | Check age → if adult, then check ID  |
| **Nested else** | Handle multiple alternative cases | Check grades → if A else if B else C |


//else if 
if (condition1) {
    // Executes if condition1 is true
} else if (condition2) {
    // Executes if condition1 is false and condition2 is true
} else if (condition3) {
    // Executes if condition1 and condition2 are false and condition3 is true
} else {
    // Executes if none of the above conditions are true
}
